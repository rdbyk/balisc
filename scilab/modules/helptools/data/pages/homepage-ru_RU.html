Добро пожаловать в Scilab 6.1

Scilab 6.1 новый главный релиз Scilab, платформы для числовых вычислений с открытым исходным кодом.

Здесь представлен тур по изменениям между Scilab 6.0.2, & 6.1.0.

Главные новые возможности

Вкратце главными новыми возможностями являются:

  • Утилиты сетевых инструментов, добавленные для использования протокола HTTP, данных JSON.
  • Значения профилирования доступны в виде значений Scilab.
  • Значения имеют более чистое отображение:
    • они отображаются в порядке присваивания
    • строки заключены в кавычки
    • комплексные числа выравнены для матриц
    • числа (IEEE 754 double) не округляются
    • полиномы и дробно-рациональные выражения отображаются более компактно
    • в структурах, встроенные списки или структуры теперь отображаются рекурсивно компактно и более чётко.
  • Добавлена функция importgui для лёгкого интерфейса csvRead

Изменения возможностей и дополнения

  • Добавлена airy: Расчёт функций Эйри первого и второго рода и их первой производной, возможно масштабирование.
  • Пустые строки используются в качестве значений по умолчанию при выделении строковых значений.
  • Добавлены функции HTTP get, post, put, upload, patch, delete.
  • Совместимость формата JSON с переменными Scilab.
  • Некорректный доступ к памяти значительно уменьшился благодаря:
    • PVS-Studio.
    • еженедельному анализу кода.
  • Усовершенствована bitget:
    • Теперь принимаются положительные знаковые кодированные целые числа.
    • Теперь поддерживаются новые типы кодированных целых чисел uint64 и int64.
    • Для десятичных чисел: теперь могут быть получены биты с индексами > 52 (вплоть до log2(number_properties("huge")) = 1024).
    • Для десятичных чисел x > 2^52, для запрошенных битов ниже %eps (индексы < log2(x)-52) теперь возвращается Nan вместо 0.
    • Теперь можно получить несколько бит для каждого элемента входного массива.
  • Усовершенствована bitset:
    • Теперь принимаются положительные знаковые кодированные целые числа.
    • Теперь поддерживаются новые типы кодированных целых чисел uint64 и int64.
    • Для десятичных чисел: теперь могут быть установлены биты с индексами > 32 (вплоть до №1024).
    • Теперь несколько битов могут быть установлены для каждого входного элемента.
    • Теперь принимаются распределённые скалярные входные данные.
  • Теперь edit принимает номер строки в виде текста (например, "23").
  • profileEnable, profileDisable, profileGetInfo могут использоваться в качестве инструментальных функций и собирать информацию во время исполнения в Scilab.
  • Усовершенствована prettyprint:
    • Теперь на самом деле поддерживаются целочисленные и текстовые входные данные.
    • Входные аргументы со значениями по умолчанию могут быть пропущены, не обязательно их указывать.
    • Результирующая строка лучше форматирована для более лёгкого переноса строк и выравнивания.
  • Введена mesh2d для вычисления двумерной сетки из векторов точек (x,y).
  • Усовершенствована loadmatfile:
    • Теперь могут быть прочитаны файлы текстовых данных, относящихся к Octave.
    • Реализовано автоматическое обнаружение формата входных данных.
    • Добавлена опция -toStruct для выдачи результатов в структуре, а не в среду вызова.
  • jremove не обязательна и clear может использоваться для освобождения Java-объектов размещённых в Scilab.
  • Введена blockdiag для улучшения, замены и расширения sysdiag в строки.
  • Усовершенствована assert_checkequal:
    • добавлена поддержка Nan и пустых значений в списках, Nan в разрежённых матрицах, явных списках, графических указателях и встроенных функциях.
    • Для массивов с совпадающими типами и размерами, и для списков одинаковой длины, сообщение об ошибке теперь указывает индексы и значения первых несовпадающих элементов.
  • Усовершенствована diophant:
    • Принимаются кодированные целые в дополнение к десятичным числам. Затем ищутся только целые решения.
    • Когда нет решения возвращается [].
    • Когда бесконечное количество решений, флаг err имеет особое значение.
    • Выходной размер получает размер на входе.
  • csvRead и csvTextScan теперь реализованы без дополнительных копий.
  • editvar GUI поддерживает копирование-вставку строк с удалением кавычек.
  • calendar теперь отображает форматированные календари.
  • xmlSetValues клонирует setPreferencesValue, который теперь устарел.
  • Усовершенствована unique:
    • возвращение числа случаев различия элементов во входном массиве.
    • возвращение разных элементов в их исходном порядке (а не в сортированном) с опцией "keepOrder".
    • все значения Nan считаются одинаковыми с опцией "uniqueNan".
  • Введена функция ellipj для вычисления эллиптических функций Якоби sn, cn, dn, ns, nc и nd.
  • Усовершенствована riccati:
    • добавлены синтаксисы riccati(H) и riccati(H, E) для описания уравнения Риккати через его гамильтониан H или пучок (E,H).
    • Ошибка прогноза возвращается в новом третьем выходном аргументе.
    • Когда решение не найдено вместо полученной ошибки возвращается X=[] | X1=[].
  • Усовершенствована hallchart:
    • Набор модулей по умолчанию устанавливается автоматически в соответствии с пределами данных.
    • Возвращаются дескрипторы сеток и меток. Их постобработка становится проще.
    • Принимаются именованные цвета; лучше цвета по умолчанию.
    • Улучшено размещение меток.
  • Усовершенствована playsnd:
    • теперь может быть определено имя wav-файла (вместо звуковых данных).
    • входной звук теперь может быть повторен заданное количество раз.
    • улучшена модальная/немодальная обработка.
  • Усовершенствована nicholschart: более нейтральный цвет рамки по умолчанию; улучшено размещение меток; теперь цвета могут указываться по их предопределённым именам или по шестнадцатеричному коду "#RRGGBB"; теперь структура дескрипторов возвращается для лёгкой постобработки как подкадров, так и набора меток.
  • sciargs возвращает столбец, а не как раньше, строку.
  • Теперь логические значения и кодированные целые числа могут конкатенироваться друг с другом, как, например, [%f int8(-5)].
  • gsort теперь может выполнять многоуровневую сортировку. Это замечательно позволяет сортировать комплексные числа.
  • factorial теперь может быть использован от n=171 вплоть до n=10^14.
  • intersect теперь поддерживает комплексные числа.
  • setdiff теперь поддерживает комплексные числа.
  • twinkle теперь может мерцать вместе с несколькими иерархически независимыми объектами, такими, как, например, кривая и её метки и т.д.
  • Усовершенствована repmat: более нейтральный цвет рамки по умолчанию; улучшено размещение меток; теперь цвета могут указываться по их предопределённым именам или по шестнадцатеричному коду "#RRGGBB"; теперь структура дескрипторов возвращается для лёгкой постобработки как подкадров, так и набора меток.
  • plot теперь работает в полу- или би-логарифмическом режиме с соответствующим входным logflag.
  • Добавлена функция plotimplicit.
  • По умолчанию более быстрое значение grid_style теперь 7, а не 3.
  • weekday принимает новые форматы дат и новые синтаксисы. Теперь доступна опция языка для применения к названиям дней.
  • Усовершенствована vectorfind:
    • Теперь vectorfind(m, []) вместо ошибки возвращает [].
    • Слишком длинная игла теперь вместо ошибки возвращает [].
    • Значения %nan никогда не совпадают. Они теперь регулярно обрабатываются как разные значения.
    • Теперь может использоваться needle короче, чем размер haystack.
    • Теперь могут указываться символ универсальной подстановки и любое значение haystack и использоваться в needle. Диапазоны фактического совпадения могут затем возвращаться: добавлены опции `joker` и `matching`.
    • Теперь в качестве haystack принимаются гиперматрицы.
    • Направление поиска теперь может быть числовым: 1, 2, ..
    • Возвращаемые индексы теперь могут быть форматированы с помощью новой опции indType.
    • Не было ни одного теста модуля. Добавлено более 100 тестов.
  • Теперь datafit может соответствовать взвешенным данным. Она теперь поддерживает любую функцию несоответствия, векторизованную для точек Data и, таким образом, она гораздо быстрее. Она теперь принимают любую матрицу параметров (не обязательно столбец). Она теперь возвращает среднее расстояние между моделью и данными и статус остановки для квазиньютоновского алгоритма.
  • Усовершенствована tree_show:
    • Новые необязательные входные данные rootTitle и styles.
    • Новая опция arrayByFields позволяет отображать каждый элемент массива как объект целиком.
    • Улучшена компоновка: детализированы индексы двумерных массивов, упрощены символы и т.д.
    • Теперь отображается объекты неявных списков и информация о функция Scilab и библиотеках функций.
  • Введена nchoosek для вычисления биномиальных коэффициентов.
  • Теперь реализованы левое .\. и правое ./. кронекеровские деления для массивов десятичных или комплексных чисел.
  • perms теперь может строить и возвращать только уникальные перестановки, без дубликатов.
  • Большинство графических объектов могут двигаться интерактивно в открытых фигурах.
  • Введена circshift.
  • atomsGetInstalledPath больше не чувствительна к регистру или полноте имён модулей. Указание версий модулей теперь необязательно.
  • function заменяет mc в качестве нового перегружающего кода для функций на языке Scilab.
