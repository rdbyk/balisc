Bienvenue dans Scilab 6.0.2

Scilab 6.0 est une nouvelle version majeure de Scilab, la plateforme open source de calcul numérique.

Voici un tour d'horizon des changements opérés entre Scilab 6.0.0 6.0.0 & 6.0.1.

Nouveautés du language

Le menu "Fichier => Ouvrir un fichier" de la console permet désormais d'ouvrir des fichiers *.xcos *.zcos *.scg ou lib avec le composant Scilab adapté, et d'autres fichiers encore avec le bon programme installé sur la machine. Tous les fichiers étaient ouverts dans SciNotes et pouvaient le planter.

"gcd" et "lcm" d'entiers retournent désormais toujours un résultat positif.

"cat" a été réécrite. Elle est plus rapide et peut traiter de larges sets de données.

Les avertissements de "fplot3d1" appelée sans option ont été retirés.

"whereis" a été améliorée :

  • Elle peut être utilisée dans les fonctions internes.
  • Quand un même nom de fonction est présent dans plusieurs librairies, "whereis" les retourne tous au lieu du dernier chargé.
  • Pour les macros utilisateurs, ""script"" est maintenant retournée au lieu de "[]".

La vitesse de "mgetl" a été augmentée pour les fichiers contenant beaucoup de lignes.

"ndgrid(x)" peut ne prendre qu'un argument d'entrée "x", avec par défaut "y=x".

Certaines fonctions ont été marquées obsolètes et seront supprimées dans Scilab 6.1 : "nanmin", "nanmax", "xgetech", "lstsize", "eval" & "dgettext" & "denom" & "numer".

Documentation

Ajout des attributs "id", "width", "height", "align" et "style" pour le tag "<imagedata>".

Ajout de l'attribut "id" pour le tag "<tr>".

De nombreuses pages one été améliorées, comprenant : "ismatrix", "xget", "xset", "ieee", "evstr", "uigetfont", "uigetdir", "uigetfile", "uiputfile", "cat", "makecell", "xstring", "norm", "barhomogenize", "colordef", "matrix", "type", "isinf", "unique", "getPreferencesValue", "gcd", "isglobal", "whereis", "mode" & "%onprompt".

Corrections de bugs

Dans cette version, de larges efforts ont été déployés pour la résolution de bugs et la stabilisation. Les informations détaillées de ces changements sont visibles sur la page CHANGES.

Ce qui était nouveau dans Scilab 6.0.x

Nouveau langage, nouveau coeur & nouvel Xcos

Cette version inclut une réécriture majeure du noyau interne de Scilab. Les modifications apportées aux aspects visibles du logiciel sont limitées.

Le premier grand changement interne est un tout nouvel analyseur syntaxique et un interprète.

Les changements de langage ne sont pas tous rétrocompatibles. Vous devez mettre à jour vos scripts/programmes/modules, ce qui devrait être très rapide dans la plupart des cas.

Le deuxième grand changement interne est la possibilité de manipuler de plus grands ensembles de données sans épuiser les ressources du système.

Xcos - l'éditeur graphique de modélisation et de simulation de systèmes dynamiques - utilise aussi le nouveau moteur de Scilab, et tourne désormais sur du code natif. Ceci devrait améliorer de manière significative les performances et permettre l'édition de schémas beacoup plus volumineux. Xcos peut charger et lancer les simulations de modèles créés avec les versions antérieures de Scilab.

Utilitaire de productivité de code

Plusieurs fonctionnalités ont été ajoutées pour permettre d'écrire plus facilement de meilleurs scripts Scilab :

  • Un debugger entièrement fonctionnel.
  • Un outil de mesure de couverture, et de profilage.
  • Un outil d’analyse statique, slint, est présent : il détecte le code peu clair, risqué, ou suspect. Il n’est pas encore documenté.

Flux d'actualité

Une nouvelle fenêtre est disponible dans l'environnement de Scilab : le "flux d'actualité". Il fournit des nouvelles, conseils et des communications générales de la communauté et de l'Équipe Scilab.

Changement de licences

Scilab est maintenant distribué selon les termes de la GNU General Public License (GPL) v2.0. Dans ses versions précédentes, Scilab était distribué selon les termes de la CeCILL v2.1 et continue d'être disponible selon ces termes.

Dans cette version, la bibliothèque Intel Math Kernel Library (MKL) et la bibliothèque FFTW optimisée sont incluses dans Scilab. Elles sont sujettes à la licence Intel (EULA).
